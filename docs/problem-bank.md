# Interview Problem Bank (C++20)

## Usage Rules

1. Always state brute-force idea first, then optimize.
2. Always write target complexity before coding.
3. Always test with edge cases (empty, one element, duplicates, max constraints).
4. Prefer STL primitives before custom data structures unless needed.

## Mock Interview Flow (45 min)

1. 5 min: clarify requirements and constraints
2. 5 min: propose approach + complexity
3. 25 min: code while narrating
4. 5 min: test with examples and edge cases
5. 5 min: discuss alternatives and trade-offs
